BlinkSSH Privacy Policy

Effective date: 16 July 2026

BlinkSSH has no account, advertising, or analytics. The developer does not collect or sell your hosts, credentials, commands, terminal content, or usage data.

Who is responsible

BlinkSSH is published by Andreas Alkstrand. For privacy questions or requests, email andreas.alkstrand@gmail.com.

Data stored on your device

BlinkSSH may store the following locally so its features work:

  • Host names, addresses, ports, usernames, groups, SSH options, and connection preferences.
  • Known-host keys and trust decisions used to detect unexpected server key changes.
  • Saved passwords, protected using Android Keystore-backed encryption.
  • References to private key files that you choose. BlinkSSH does not upload those keys.
  • Terminal, theme, keyboard, clipboard, autocomplete, and accessibility preferences.
  • Command snippets and recent command history used for local autocomplete.
  • Optional on-device AI model files and their installation state.

Network connections

When you connect, BlinkSSH sends authentication data, commands, terminal input, and related SSH traffic to the host, proxy, jump host, or forwarded destination that you configured. SSH traffic is encrypted between your device and the relevant SSH endpoint. Andreas Alkstrand does not operate these endpoints and cannot read that traffic.

A connection test may resolve the host, open a TCP connection, negotiate SSH, inspect the server host key, and check whether the configured authentication can proceed. It does not run a shell command on the server.

If you choose an optional AI autocomplete model, Android downloads the model over HTTPS from Hugging Face. Hugging Face may receive standard connection information such as your IP address under its own privacy practices. The model then runs locally; BlinkSSH does not send terminal context to an AI service.

Clipboard and files

BlinkSSH accesses clipboard content when you explicitly copy or paste. Remote clipboard writes using OSC 52 are disabled unless you enable them. Files selected for SSH keys or imports are read only for the action you request and are not sent to the developer.

Data collection and sharing

BlinkSSH does not include advertising, analytics, crash-reporting, or developer telemetry services. The developer does not collect or share personal data from the app. User-directed SSH transfers go only to the destinations you choose.

Security, backup, and exports

Android app backup is disabled. Saved passwords are encrypted using keys held by Android Keystore. BlinkSSH host-profile exports omit saved passwords, private key material, and known-host trust records. No software can guarantee absolute security, so protect exported files and your device.

Retention and deletion

Local data remains until you delete the relevant host, password, history, model, or app data. Uninstalling BlinkSSH removes its private app data. Files you deliberately exported remain wherever you saved or shared them and must be deleted separately.

Children

BlinkSSH is a technical tool for managing computers and is not directed to children under 13.

Changes

This policy may be updated when BlinkSSH's features or legal obligations change. The effective date above identifies the current version.
